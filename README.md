# Car Sorting Project

Учебный консольный проект для работы с данными об автомобилях. Реализует хранение в кастомных структурах данных, алгоритмическую обработку и многопоточный анализ.

## Особенности
* **Кастомная коллекция**: реализация `MyArrayList` на базе динамического массива с поддержкой интерфейса `List`.
* **Сортировка**: реализация паттерна Strategy для алгоритмов Bubble Sort, Selection Sort, Insertion Sort и Quick Sort.
* **Ввод данных**: заполнение коллекций через Stream API тремя методами: из JSON-файла, пользовательского ввода или в случайном порядке.
* **Многопоточность**: метод параллельного поиска машин по их мощности, сравнение с однопоточным поиском.
* **Валидация**: все данные валидируются перед попаданием в список.

## Применяемые паттерны
* **Strategy**: алгоритмы сортировки назначаются и исполняются через инкапсулирующий стратегию сервис.
* **Builder**: более гибкое создание объектов `Car` с помощью Builder.
* **Chain of Responsibility**: валидация объектов сущности через цепочку обязанностей (по очереди всех полей).

## Пакеты
* `org.example.exception` — классы кастомных исключений.
* `org.example.input` — реализации чтения данных с использованием Java Streams.
* `org.example.list` — кастомная реализация списка.
* `org.example.model` — сущность автомобиля с набором компараторов.
* `org.example.service` — сервис, инкапсулирующий стратегию сортировки.
* `org.example.sorting` — реализации алгоритмов сортировки.
* `org.example.ui` — пользовательский интерфейс.
* `org.example.util` — пакет для утилит, содержит многопоточный поиск.
* `org.example.validation` — валидация класса по цепочке ответственности.
